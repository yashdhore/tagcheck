01
Config intake
Human-in-the-loop

Client's expected tag rules get turned into a config file — what should fire, where, and under what conditions. This is what makes the tool reusable across clients instead of Vail-specific.

tag_rules.yaml Adobe Launch export naming convention map expected events / page types
02
Site crawl & capture
Automated

Headless crawler walks the site (sitemap or defined page list), and for every page captures the network layer — every tag call actually fired, not just what's configured in Launch.

Crawl engine

Playwright/Puppeteer headless browser, page-by-page, respects auth walls and geo rules.

Capture layer

Network request interception — logs every beacon, pixel, and dataLayer push per page.

03
Rule engine — audit logic
Automated

Captured tags get diffed against the config from Step 1. This is the core IP — the same audit methodology your subcontractor built for Vail, generalized into reusable rule checks.

missing tags duplicate fires malformed variables naming violations load-order issues
04
Analyst review
Human-in-the-loop

Raw findings get triaged — false positives filtered out, severity assigned, and findings annotated with plain-language explanations. This is the "human-reviewed" part of your pitch, and it's what differentiates you from a raw ObservePoint export.

05
Client report — Streamlit dashboard
Client-facing output

Delivered as a live Streamlit app: pass/fail by page and rule, trend view for repeat quarterly runs, and an exportable PDF summary for stakeholders who won't open the dashboard.
